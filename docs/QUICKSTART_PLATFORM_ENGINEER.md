# Quick Start: Platform Engineer

**Time to complete:** 15 minutes

## What You'll Do

Create a new RGD (Resource Graph Definition) for a Redis cache - from template development to deployment via GitOps.

## Prerequisites

Before you begin:

1. **Development environment** - Local machine with kubectl access
2. **ytt installed** - [Install Carvel ytt](https://carvel.dev/ytt/) for templating
3. **flux CLI installed** - [Install Flux CLI](https://fluxcd.io/flux/installation/) for artifact building
4. **Git access** - Write access to fedCORE platform repository
5. **Basic understanding** - Familiarity with Kubernetes, ytt templating, and Kro RGDs

**Verify prerequisites:**

```bash
# Check ytt is installed
ytt version

# Check flux is installed
flux version

# Check kubectl access
kubectl cluster-info
```

## Step 1: Create Base Template Directory Structure

Create the directory structure for your new RGD:

```bash
# Navigate to RGDs directory
cd platform/rgds/

# Create RGD directory structure
mkdir -p redis-cache/{base,overlays/{aws,azure,onprem}}

# Create necessary files
touch redis-cache/base/{kro-schema.yaml,deployment.yaml,service.yaml,configmap.yaml}
touch redis-cache/overlays/aws/{elasticache.yaml,iam-role.yaml}
touch redis-cache/overlays/azure/{azure-cache.yaml,managed-identity.yaml}
touch redis-cache/overlays/onprem/{redis-statefulset.yaml,pvc.yaml}
```

**Directory structure:**

```
platform/rgds/redis-cache/
├── base/                        # Cloud-agnostic resources
│   ├── kro-schema.yaml         # Kro RGD definition
│   ├── deployment.yaml          # Redis client deployment
│   ├── service.yaml             # Service for Redis
│   └── configmap.yaml           # Redis configuration
├── overlays/
│   ├── aws/                     # AWS-specific resources
│   │   ├── elasticache.yaml    # ElastiCache cluster
│   │   └── iam-role.yaml        # IAM role for access
│   ├── azure/                   # Azure-specific resources
│   │   ├── azure-cache.yaml    # Azure Cache for Redis
│   │   └── managed-identity.yaml
│   └── onprem/                  # On-premises resources
│       ├── redis-statefulset.yaml
│       └── pvc.yaml             # Persistent storage
└── README.md                    # Documentation
```

## Step 2: Define Kro Schema

Create the RGD schema in `base/kro-schema.yaml`:

```yaml
apiVersion: kro.run/v1alpha1
kind: ResourceGroup
metadata:
  name: redis-cache
spec:
  # User-facing API schema
  schema:
    apiVersion: v1alpha1
    kind: RedisCache
    spec:
      # Basic configuration
      name: string
      namespace: string

      # Redis configuration
      version: string | default="7.0"
      replicas: number | default=2
      nodeType: string | default="cache.t3.micro"

      # Memory and eviction
      maxMemory: string | default="1gb"
      evictionPolicy: string | default="allkeys-lru"

      # Network and security
      port: number | default=6379
      tls:
        enabled: bool | default=true

      # High availability
      clusterMode:
        enabled: bool | default=false
        shards: number | default=1

      # Backup and persistence
      backup:
        enabled: bool | default=true
        retentionDays: number | default=7

  # Resources generated by this RGD
  resources:
    - id: redis-config
      template:
        apiVersion: v1
        kind: ConfigMap
        metadata:
          name: ${schema.spec.name}-config
          namespace: ${schema.spec.namespace}
        data:
          redis.conf: |
            maxmemory ${schema.spec.maxMemory}
            maxmemory-policy ${schema.spec.evictionPolicy}
            port ${schema.spec.port}

    - id: redis-service
      template:
        apiVersion: v1
        kind: Service
        metadata:
          name: ${schema.spec.name}
          namespace: ${schema.spec.namespace}
        spec:
          selector:
            app: ${schema.spec.name}
          ports:
            - port: ${schema.spec.port}
              targetPort: ${schema.spec.port}

    - id: redis-secret
      template:
        apiVersion: v1
        kind: Secret
        metadata:
          name: ${schema.spec.name}-credentials
          namespace: ${schema.spec.namespace}
        stringData:
          # Placeholder - will be populated by cloud provider
          endpoint: ""
          port: "${schema.spec.port}"
          password: ""

    # Status fields exposed to users
    status:
      endpoint: ${redis-service.spec.clusterIP}
      port: ${schema.spec.port}
      ready: ${redis-service.status.ready}
```

## Step 3: Create Cloud Overlays

### AWS Overlay (ElastiCache)

Create `overlays/aws/elasticache.yaml`:

```yaml
#@ load("@ytt:data", "data")
#@ load("@ytt:overlay", "overlay")

#@overlay/match by=overlay.all
---
apiVersion: kro.run/v1alpha1
kind: ResourceGroup
metadata:
  name: redis-cache
spec:
  #@overlay/append
  resources:
    # ElastiCache cluster
    - id: elasticache-cluster
      template:
        apiVersion: elasticache.services.k8s.aws/v1alpha1
        kind: CacheCluster
        metadata:
          name: #@ data.values.name
          namespace: #@ data.values.namespace
        spec:
          cacheNodeType: #@ data.values.nodeType
          engine: redis
          engineVersion: #@ data.values.version
          numCacheNodes: #@ data.values.replicas
          port: #@ data.values.port

          # Security
          transitEncryptionEnabled: #@ data.values.tls.enabled
          atRestEncryptionEnabled: true
          authTokenEnabled: true

          # High availability
          preferredMaintenanceWindow: "sun:05:00-sun:06:00"
          snapshotRetentionLimit: #@ data.values.backup.retentionDays

          # IAM authentication
          serviceAccountName: #@ data.values.name + "-sa"

    # ServiceAccount with IAM role
    - id: redis-serviceaccount
      template:
        apiVersion: v1
        kind: ServiceAccount
        metadata:
          name: #@ data.values.name + "-sa"
          namespace: #@ data.values.namespace

    # IAM role for Redis access
    - id: redis-iam-role
      template:
        apiVersion: iam.services.k8s.aws/v1alpha1
        kind: Role
        metadata:
          name: #@ data.values.name + "-role"
          namespace: #@ data.values.namespace
        spec:
          name: #@ "fedcore-" + data.values.namespace + "-" + data.values.name
          assumeRolePolicyDocument: |
            {
              "Version": "2012-10-17",
              "Statement": [{
                "Effect": "Allow",
                "Principal": {"Service": "pods.eks.amazonaws.com"},
                "Action": "sts:AssumeRole"
              }]
            }
          policies:
            - policyName: redis-access
              policyDocument: |
                {
                  "Version": "2012-10-17",
                  "Statement": [{
                    "Effect": "Allow",
                    "Action": [
                      "elasticache:DescribeCacheClusters",
                      "elasticache:DescribeReplicationGroups"
                    ],
                    "Resource": "*"
                  }]
                }
```

### Azure Overlay (Azure Cache for Redis)

Create `overlays/azure/azure-cache.yaml`:

```yaml
#@ load("@ytt:data", "data")
#@ load("@ytt:overlay", "overlay")

#@overlay/match by=overlay.all
---
apiVersion: kro.run/v1alpha1
kind: ResourceGroup
metadata:
  name: redis-cache
spec:
  #@overlay/append
  resources:
    # Azure Cache for Redis
    - id: azure-redis
      template:
        apiVersion: cache.azure.com/v1beta1
        kind: Redis
        metadata:
          name: #@ data.values.name
          namespace: #@ data.values.namespace
        spec:
          location: eastus
          sku:
            name: Basic
            family: C
            capacity: 1
          enableNonSslPort: false
          minimumTlsVersion: "1.2"
          redisConfiguration:
            maxmemory-policy: #@ data.values.evictionPolicy

    # Managed Identity for Redis access
    - id: redis-managed-identity
      template:
        apiVersion: managedidentity.azure.com/v1beta1
        kind: UserAssignedIdentity
        metadata:
          name: #@ data.values.name + "-identity"
          namespace: #@ data.values.namespace
```

### On-Premises Overlay (Redis StatefulSet)

Create `overlays/onprem/redis-statefulset.yaml`:

```yaml
#@ load("@ytt:data", "data")
#@ load("@ytt:overlay", "overlay")

#@overlay/match by=overlay.all
---
apiVersion: kro.run/v1alpha1
kind: ResourceGroup
metadata:
  name: redis-cache
spec:
  #@overlay/append
  resources:
    # Redis StatefulSet
    - id: redis-statefulset
      template:
        apiVersion: apps/v1
        kind: StatefulSet
        metadata:
          name: #@ data.values.name
          namespace: #@ data.values.namespace
        spec:
          serviceName: #@ data.values.name
          replicas: #@ data.values.replicas
          selector:
            matchLabels:
              app: #@ data.values.name
          template:
            metadata:
              labels:
                app: #@ data.values.name
            spec:
              containers:
                - name: redis
                  image: #@ "redis:" + data.values.version
                  ports:
                    - containerPort: #@ data.values.port
                  volumeMounts:
                    - name: redis-data
                      mountPath: /data
                    - name: redis-config
                      mountPath: /usr/local/etc/redis
                  resources:
                    requests:
                      cpu: 100m
                      memory: 256Mi
                    limits:
                      cpu: 1000m
                      memory: #@ data.values.maxMemory
          volumeClaimTemplates:
            - metadata:
                name: redis-data
              spec:
                accessModes: ["ReadWriteOnce"]
                resources:
                  requests:
                    storage: 10Gi
```

## Step 4: Test Locally with fedcore build

Build and validate your RGD locally:

```bash
# Build the RGD artifact for an AWS cluster
fedcore build \
  --artifact platform/rgds/redis-cache \
  --cluster platform/clusters/fedcore-dev-use1

# Check the generated artifact
ls -lh /tmp/artifacts/redis-cache-*.tar.gz

# Extract and inspect the artifact
mkdir -p /tmp/redis-cache-test
tar -xzf /tmp/artifacts/redis-cache-*.tar.gz -C /tmp/redis-cache-test
cat /tmp/redis-cache-test/manifests/*.yaml
```

**Validation checklist:**
- [ ] All YAML is valid (no syntax errors)
- [ ] Cloud-specific resources included (ElastiCache for AWS)
- [ ] Kro schema is well-formed
- [ ] ytt overlays applied correctly

**Test with a sample manifest:**

Create `test-redis.yaml`:

```yaml
apiVersion: v1alpha1
kind: RedisCache
metadata:
  name: test-cache
  namespace: default
spec:
  version: "7.0"
  replicas: 2
  nodeType: cache.t3.micro
  maxMemory: 2gb
  evictionPolicy: allkeys-lru
```

Apply to test cluster:

```bash
kubectl apply -f test-redis.yaml
kubectl get rediscache test-cache --watch
```

## Step 5: Enable in Cluster Configs

Add your RGD to cluster configurations:

**Edit `platform/clusters/fedcore-dev-use1/enabled-rgds.yaml`:**

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: enabled-rgds
  namespace: flux-system
data:
  rgds: |
    - webapp
    - database
    - queue
    - bucket
    - redis-cache  # New RGD
```

**Create Flux OCI Repository for RGD:**

Edit `platform/clusters/fedcore-dev-use1/flux-sources.yaml`:

```yaml
apiVersion: source.toolkit.fluxcd.io/v1
kind: OCIRepository
metadata:
  name: redis-cache
  namespace: flux-system
spec:
  interval: 10m
  url: oci://nexus.fedcore.io/platform/redis-cache
  ref:
    semver: ">=1.0.0"
---
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: redis-cache
  namespace: flux-system
spec:
  interval: 10m
  sourceRef:
    kind: OCIRepository
    name: redis-cache
  path: ./
  prune: true
  wait: true
```

## Step 6: Deploy via GitOps

Commit and push your changes:

```bash
# Add all new RGD files
git add platform/rgds/redis-cache/

# Add cluster configuration changes
git add platform/clusters/fedcore-dev-use1/

# Commit with descriptive message
git commit -m "Add Redis Cache RGD with multi-cloud support

- Implemented base Kro schema with Redis configuration
- Added AWS overlay with ElastiCache provisioning
- Added Azure overlay with Azure Cache for Redis
- Added on-prem overlay with Redis StatefulSet
- Enabled RGD in fedcore-dev-use1 cluster
- Tested locally with sample manifest"

# Push to trigger CI/CD
git push origin main
```

**What happens next:**

1. **CI/CD Pipeline Runs:**
   - Validates all YAML syntax
   - Runs security scans (Kyverno policies)
   - Builds OCI artifacts per cluster
   - Pushes artifacts to Nexus registry

2. **Flux Syncs to Cluster:**
   - Detects new OCI artifact
   - Pulls artifact from Nexus
   - Applies RGD definition to cluster
   - Kro controller registers new RedisCache CRD

3. **RGD Available:**
   - Developers can now create RedisCache resources
   - Kro orchestrates provisioning based on cloud overlay

**Monitor deployment:**

```bash
# Watch Flux reconciliation
flux get kustomizations --watch

# Check if RGD is registered
kubectl get resourcegroups redis-cache

# Verify CRD is created
kubectl get crd rediscaches.v1alpha1.platform.fedcore.io

# Check Kro controller logs
kubectl logs -n kro-system -l app=kro-controller --tail=100
```

## Troubleshooting

### ytt build fails

Check ytt syntax:
```bash
ytt -f platform/rgds/redis-cache/base/ \
    -f platform/rgds/redis-cache/overlays/aws/ \
    --data-values-file platform/clusters/fedcore-dev-use1/values.yaml
```

Common issues:
- Missing `#@ load` directives
- Incorrect overlay match expressions
- Type mismatches in data values

### Kro doesn't register RGD

Check Kro controller logs:
```bash
kubectl logs -n kro-system -l app=kro-controller --tail=100
```

Verify ResourceGroup syntax:
```bash
kubectl describe resourcegroup redis-cache
```

Common issues:
- Invalid schema syntax
- Circular dependencies between resources
- Missing required fields

### Cloud resources not provisioning

Check ACK/ASO controller logs:
```bash
# AWS (ACK)
kubectl logs -n ack-system -l app.kubernetes.io/name=elasticache-controller

# Azure (ASO)
kubectl logs -n azureserviceoperator-system -l control-plane=controller-manager
```

Verify IAM permissions:
```bash
# Check Pod Identity association
kubectl get podidentityassociation -n ack-system
```

Common issues:
- Missing IAM permissions in tenant account
- Incorrect trust policy on cross-account role
- Permission boundary blocking resource creation
